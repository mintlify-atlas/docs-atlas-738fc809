  {/* Feature Grid */}
  <CardGroup cols={3}>
    <Card title="Zero overhead" icon="bolt">
      Uses `Bun.serve` directly with the standard `Request`/`Response` model. No hidden middleware chains, no magic routing.
    </Card>
    <Card title="Structured routing" icon="sitemap">
      File-based modules under `src/routes/name/` mounted at `/api/v1/name`. First match wins with explicit path and method checks.
    </Card>
    <Card title="Rate limiting" icon="shield">
      Per-route in-memory limiters via `rate-limiter-flexible`. Independent quotas per endpoint â€” no global bottleneck.
    </Card>
    <Card title="Security headers" icon="lock">
      HSTS, CSP, X-Frame-Options, Permissions-Policy, and more applied automatically to every response.
    </Card>
    <Card title="Request correlation" icon="link">
      Each request gets a unique `rid` logged on every line and mirrored via `X-Request-ID` using `AsyncLocalStorage`.
    </Card>
    <Card title="Observability" icon="chart-line">
      In-memory metrics, distributed tracing with `X-Trace-ID`, structured logging, and client IP resolution built in.
    </Card>
    <Card title="Body-size limits" icon="weight-scale">
      Per-route `Content-Length` enforcement with configurable ceilings â€” environment default or per-endpoint override.
    </Card>
    <Card title="CORS middleware" icon="globe">
      Configurable origin allowlist with preflight handling. Deny-by-default in production unless `CORS_ALLOWED_ORIGINS` is set.
    </Card>
    <Card title="Input validation" icon="check-circle">
      Zod-based helpers for body, query, and route params. Pre-built schemas for pagination, IDs, and email addresses.
    </Card>
  </CardGroup>
